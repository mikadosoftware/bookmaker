LaTeX and ReST

LaTeX is an immensely powerful typesetting package. It is not a word processor. It takes "plain" text and it works out the best way to present that text as a book or report or article. It is used to produce truly beautiful layout, and more importantly, it allows, no, it makes you focus on the words, on what you are going to say. I find it in many ways superior to any word processing package, except for one thing. It is hard to get started.

So here is my guide to getting started with LaTeX.

Firstly a list of reasons why to use LaTeX. Its always good to have reasons to sustain you through the dark times.

  • LaTeX is used by some of the cleverest people I know, and some of the cleverest people on the planet. It is to all intents and purposes the language scientists learn to write their doctoral thesis, and publish their papers in journals.
  • If LaTeX had been around in 1905, Einstein would have used it to typeset his theories on relativity, nature of light and the size of atoms. Of course he would have had to take six months to learn it and then get that last annoying tweak out and a revolution in science would have had to wait another year, but he still would have used LaTeX. [1]
  • It is based on TeX, which was written by Prof. Donald Knuth 30 years ago, and it is still going strong.
  • TeX has a version numbering system that is tending to pi - in other words we are at version 3.14152. If that is not cool I do not know what is.
  • It is pronounced Lay-Tech.

and some more serious reasons

  • You write a text file, and concentrate on the meaning. LaTeX makes you look good.
  • The output looks better. It just does. And you can make it even better. You.

Ok, so there is some clever LaTeX language which does typesetting. But writing LaTeX is as horrible as writing HTML. Yes, this is true, and luckily I do not do that. Some clever people have written a Python suite called docutils, which basically takes text that is formatted in consistent manner, and can turn that into HTML or LaTeX. Then you can turn that LaTeX into a pdf.

I use ReST, which is a "lite" markup language. This whole site was written in it.

This is a Title in ReST
=======================
This is just a paragraph, and below is a bullet list

- with bullet points
- laid out easily
- in any text editor.

and you can even include images.

ReST is exceptionally flexible, and you can produce HTML at the drop of a hat:

$ rst2html.py --stylesheet=style.css test.txt > test.html

And of course it can do the same with LaTeX:

$ rst2latex.py test.txt > test.ltx

Which you can then convert into a pdf:

$ pdflatex test.ltx

OK, we can take a simple piece of text, run it into LaTeX, run the LaTeX into PDF. But can we make it look, well, more corporatey.

Syling

Frankly only a PhD dissertation committee could love that output. It is just not, well, jazzy enough these days. A lot of LaTeX purists (who also happen to be leading experts in typesetting and information presentation, but what do they know), feel that the over-styling of modern documents tends to lose the content being discussed. This is probably true but if you work in almost any business these days, they have a corporate style, and you will have no end of trouble fighting it so go with the flow.

My company had requirements for 12pt arial, title pages, headers and footers etc. But I have this nice workflow that produces pdf outputs - how can I get the corporate styles I need without hand editing the LaTeX documents.

Stylesheets for LaTeX

This is where .sty documents come in. A .sty document is called a Package in LaTeX, but obviously has its roots in style (hence the .sty). A LaTeX style file can be thought of like CSS for HTML. Although .sty files also provide real functionality (the graphicx package is hugely complicated), for our purposes just getting to 12pt Arial is enough.

So I need to include this line in my LaTeX document [2]

\usepackage{pbrian}

and suddenly, with no other changes to the LaTeX, the document looks much better

Corporate Style

Which is of course exactly what I use this stuff for. Mostly producing business reports and other "corporate" documents. It means I can concentrate on the words, and not have to fuss around with making it look nice.

It feels deeply effective to be able to write just some plain vanilla text, hit a command and view a professional looking, impressive document that can be mailed to whoever needs it. Quotes, proposals, tenders, political in-fighting. All of which I can concentrate on the content of the document, and then let ReST and LaTeX make me look good.

Of course one can use MS Word (or any word processor) effectively and well,[#]_ but I have found this a indispensible tool. Often I can find myself where Word is impractical (say on the tube with just a PDA) and just write an email to myself, in plain text, confident that when i am in front of a real computer, it will be a matter of minutes to turn it into a professional document.

Getting hold of LaTeX

Writing Own class

There are two types of file that contain user-defined style information - classes (.cls) and packages (.sty). A package can be included with any kind of class (article, book) and it will work sensibly. It usually includes things like author information. A class is much harder to work with other classes - a book class will work badly if it tries to coexist with a newsletter class.

We want a business-report class, that we shall use to define how all the memos and reports we send out to the business shall look. It will try and mimic a Word document as corporate style usually gets defined like that.

Notes

difference between class and pacakge

classes are in .cls files, and only do one type of document (ie newsletter) packages are in .sty files and have functions that can affect any type of document, such as graphics package.
